                                  如何利用XFTP部署你的网站

                                  XinYang's Blog大约 4 分钟运维
                                  此页内容
                                  • 准备打包后的网站

                                  通过此文章，你将学会如何利用XFTP部署你的静态网站。

                                  可以用作部署Web的服务器有很多，如Tomcat、Nginx，Node、Apache、Cloudflare、OpenResty等

                                  这里我们以常用的Nginx举例，Nginx适用于多个HTML构成的网站、或者通过webpack、vite等构建工具打包后的网站，不适用于部署使用。

                                  在了解此文章之前，你需要具备的前置知识：

                                  1. 了解基本的Centos操作命令
                                  2. 如何使用XSHELL远程连接服务器
                                  3. nginx的基本使用
                                  4. 如何使用在Centos上安装nginx
                                  5. 安装好XFTPopen in new window

                                  若了解如何部署SSR框架打包构建的网站，如Nuxt、Next，则需要了解的前置知识：Centos安装node

                                  准备打包后的网站

                                  # step 1
npm init vite react-app -- --template react

# step 2
cd react-app && npm install

# step 3 
npm run build

                                  以上命令构建了一个react框架打包后的项目

                                  打包SSR框架构建的网站

                                  学习此小节请确保你的服务器安装了node，如果没有你可以查看此教程。

                                  这里我选择初始化Nuxt3open in new window项目。但是由于官方给的方法初始化项目会失败，可以在此直接下载模板open in new window

                                  或者进入此网页open in new window下载代码，即为Nuxt3模板。

                                  1. 安装依赖并打包
                                  # step 1 确保node版本大于16.10.0
node -v

# step 2 进入你下载的nuxt3文件目录，安装依赖，这个过程会非常漫长，需要耐心等待
npm install

# step 3 安装好后依赖，进行打包
npm run build
                                  1. 按以上步骤完成后，可以看到项目根目录下多个一个.output文件夹，进入该文件夹，在此文件夹新建ecosystem.config.js，并添加以下代码：
                                  module.exports = {
  apps: [
    {
      name: "NuxtAppName",// 进程名
      port: "80",// 进程端口号
      exec_mode: "cluster", // 运行模式
      // instances: "max", // 分配的线程，会自动根据CPU的内核数量进行分配
      script: "././server/index.mjs", //运行的js文件路径。
    },
  ],
};
